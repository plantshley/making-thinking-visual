üìäParameters by Event Type

üìÑpage_view

Tracks all page visits (hub + 8 detail pages)

page_name Title of the page visited (e.g., "Creating a Repository", "GitHub Setup Starmap Hub")
page_location Full URL of the page
page_path Path portion of the URL (e.g., "/pages/creating-repo.html")

üó∫Ô∏ènode_clicked

Tracks clicks on roadmap nodes (both desktop SVG and mobile grid)

node_name Name of the clicked node (e.g., "GitHub Desktop", "Creating a Repository")
navigation_type Either "desktop" (SVG roadmap) or "mobile" (grid navigation)

üåôtheme_changed

Tracks theme toggle between dark and light modes

theme_selected The new theme chosen: "dark" or "light"
previous_theme The theme that was active before the change

‚ôøfont_accessibility_toggled

Tracks when users enable/disable accessible fonts (OpenDyslexic)

accessible_fonts_enabled Boolean: true (enabled) or false (disabled)

üìëtab_clicked

Tracks tab navigation in multi-option pages (workflow choices, methods, etc.)

tab_group Section or page ID where the tab exists (e.g., "remote", "local", "customization")
tab_name Name of the clicked tab (e.g., "GitHub Desktop", "VS Code", "Desktop ‚Üí VS Code")

üîóexternal_link_clicked

Tracks clicks on external resource links (GitHub.com, VS Code, etc.)

link_platform Platform category: "github", "vscode", "github-desktop", "anthropic", or "external"
link_url Full URL of the clicked link
link_label Text displayed for the link

‚Ü©Ô∏èback_to_map_clicked

Tracks when users click "Back to Map" links to return to the hub

source_page The page they're leaving to return to map (e.g., "Creating a Repository")

‚≠êCustom Dimension Priority List

When registering custom dimensions in GA4, prioritize based on analytical value:

High Priority Register First

  • node_name - Core navigation tracking
  • page_name - Essential for page analytics
  • navigation_type - Desktop vs mobile usage patterns
  • tab_name - Workflow preference insights
  • tab_group - Contextual workflow analysis
  • theme_selected - User preference data

Medium Priority Register Second

  • accessible_fonts_enabled - Accessibility usage metrics
  • link_platform - External resource engagement
  • source_page - Navigation flow patterns
  • previous_theme - Theme switching behavior

Low Priority Optional

  • link_url - Detailed link tracking (use link_platform instead for most analyses)
  • link_label - Link text details
  • page_path - Redundant with page_name
  • page_location - Full URL (usually unnecessary)

‚öôÔ∏èHow to Register Custom Dimensions in GA4

Important: Custom dimensions must be registered in GA4 before they'll appear in reports. You have a limit of 50 event-scoped custom dimensions per property.

Step-by-Step Process:

  1. Go to Admin (gear icon in bottom left)
  2. Under the Property column, click Custom definitions
  3. Click Create custom dimension
  4. Fill in the form:
    • Dimension name: Human-readable name (e.g., "Node Name")
    • Scope: Select Event
    • Event parameter: Exact parameter name from code (e.g., "node_name")
    • Description: Optional note about what it tracks
  5. Click Save
  6. Repeat for each parameter you want to analyze
Pro Tip: Start with the High Priority dimensions listed above. You can always add more later as needed!
