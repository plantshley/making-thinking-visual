üíêCustom Parameters by Event Type

ü¶Ñbody_selected Event

Tracks when a user selects a body type in the creature builder

body_type The ID of the selected body (e.g., "cat", "bunny", "dragon")
body_name The display name of the body (e.g., "Cat", "Bunny", "Dragon")

üíæcreature_exported Event

Tracks when a user downloads their created creature

object_count Number of objects placed on the creature
has_drawing Whether the creature includes free-draw lines (true/false)
has_body Whether a body type was selected (true/false)

‚úèÔ∏èdrawing_mode_enabled Event

Tracks when a user activates the free draw mode

- No custom parameters (simple event trigger)

üíæcreature_saved Event

Tracks when the creature is auto-saved to localStorage

object_count Number of objects in the saved creature
has_body Whether a body type is saved (true/false)
has_drawing Whether drawing lines are saved (true/false)

üìÑpage_view Event

Tracks when a user navigates to a different page/tab

page_name Name of the page viewed (home, build, shop, gallery, links)
page_location Full URL of the page
page_path Path portion of the URL

üé®theme_changed Event

Tracks when a user switches the site theme

theme_id ID of the selected theme (e.g., "twinkleFairyDream", "glitterGroovyRainbow")
theme_name Display name of the theme (e.g., "Twinkle Fairy Dream", "Glitter Groovy Rainbow")

‚ôøaccessible_fonts_toggled Event

Tracks when a user toggles the accessible fonts feature

enabled Whether accessible fonts are now enabled (true/false)

üîósocial_link_clicked Event

Tracks when a user clicks an external social/shop link

link_label Label of the link (e.g., "Instagram", "Venmo", "Ko-fi Shop")
link_url Full URL of the destination
link_platform Platform identifier (e.g., "instagram", "venmo", "ko-fi_shop")

üìÇgallery_category_selected Event

Tracks when a user clicks a gallery category

category_id ID of the selected category
category_name Display name of the category
source Where the category was clicked from (e.g., "tab_navigation")

üñºÔ∏ègallery_image_clicked Event

Tracks when a user clicks to view a gallery image

category_id ID of the category containing the image
category_name Name of the category containing the image
image_path Path to the clicked image
image_index Position of the image in the category grid (0-based)

‚è±Ô∏ètiming_complete Event

Tracks time-based engagement metrics

name Name of the timing metric (e.g., "session_duration", "shop_page_time", "home_page_time")
value Time value in seconds
event_category Category of the timing event (e.g., "engagement", "page_engagement", "shop_engagement")

üéÄHow to Register Custom Dimensions

  1. Go to Admin (gear icon in bottom left of Google Analytics)
  2. In the Property column, click Custom definitions
  3. Click Create custom dimension
  4. For each parameter:
    • Dimension name: Use a friendly name (e.g., "Body Type")
    • Scope: Select "Event"
    • Description: Add a brief description
    • Event parameter: Enter the exact parameter name (e.g., "body_type")
  5. Click Save
  6. Repeat for each parameter you want to track
üíê Important: Custom dimensions can take 24-48 hours to start collecting data. Events sent BEFORE you create the dimension won't be retroactively updated, but all new events will include the dimension automatically.

Timing Events Explained:

The timing_complete event tracks user engagement through time-based metrics:

  • session_duration - Total time from site entry to exit (tracked on page close/unmount)
  • {page}_page_time - Time spent on each individual page (home, build, shop, gallery, links)
  • All timing values are in seconds
  • Tracked using React useRef to avoid re-renders
